Pertemuan 4:
Sistem Nyawa (Health System)

Membuat Game Menjadi Menantang!

Halo, Game Developer! Siap naik level hari ini?

Misi Kita Hari Ini!

Misi Utama:
Membuat sistem NYAWA atau HEALTH.

Jika Player kita menyentuh Zombie, nyawanya akan berkurang. Ini adalah inti dari sebuah game survival!

Ingat Lagi (Review)

"Ada yang masih ingat, apa fungsi dari blok `delete this clone`?"

Untuk menghapus kloning! Misalnya saat peluru kena zombie, atau zombie sudah dikalahkan. Ini penting agar game tidak 'lag'.

Konsep Utama: VARIABEL

Ada yang pernah pakai Variable?
Variabel itu seperti "kotak ajaib" untuk menyimpan data.

Data yang paling sering kita simpan adalah Skor atau Waktu.

Variabel Bukan Cuma Skor!

Di level SMP, kita akan pakai Variabel untuk hal yang lebih keren!
Variabel bisa menyimpan data APAPUN, misalnya:

Jumlah Koin

Sisa Peluru

Dan tentu saja... NYAWA!

Blok Fokus: Variabel

Ini adalah 3 blok utama untuk mengelola Variabel. Kita akan gunakan semuanya hari ini.

Tantangan Mini: Buat Variabel 'Nyawa'

  1. Klik kategori Variables (Oranye Tua).
  2. Klik tombol "Make a Variable".
  3. Beri nama variabel baru kita: Nyawa
  4. Pastikan kamu memilih "For all sprites".
  5. Klik OK.

Lihat! Sekarang ada tampilan 'Nyawa' di layar game kita!

Misi 1: Inisialisasi Nyawa

"Inisialisasi" adalah istilah keren untuk "Pengaturan Awal".

Setiap kali game dimulai, nyawa pemain harus penuh, kan?

Kira-kira, berapa nyawa awal kita? (Misalnya: 3)

Langkah 1: Set Nyawa Awal

Ayo pindah ke sprite Karakter Utama (misal: Player atau Hero).
Kita tambahkan pengaturan ini di skrip when green flag clicked.

Bedanya 'Set' vs 'Change'?

Penting! Jangan sampai tertukar!

SET [Nyawa] to 3

"Atur nilainya JADI 3." (Seperti mengganti isi kotak)
Digunakan di AWAL permainan.

CHANGE [Nyawa] by -1

"Ubah nilai yang ada, dikurangi 1." (Seperti mengambil 1 koin dari kotak)
Digunakan SAAT tertabrak.

Misi 2: Deteksi Tabrakan!

Bagian paling penting!
Bagaimana caranya Player tahu kalau dia menyentuh Zombie?

Kita butuh SENSOR! Blok apa yang kita pakai?

Sensor 'Touching'

Betul! Kita pakai touching...? (Menyentuh...?).
Dan kita masukkan ke dalam blok if...then.

Kita harus mengecek ini TERUS MENERUS.
Jadi, kodenya kita taruh di mana?

Langkah 2: Cek Tabrakan

Di dalam "Game Loop" (blok forever) di sprite Player.

Artinya: Selamanya, jika menyentuh sprite Zombie, maka... (apa yang terjadi kalau tabrakan?)

Langkah 3: Mengurangi Nyawa

Jika menyentuh Zombie... maka nyawa berkurang!
Ayo tambahkan blok change di dalamnya.

Mari Kita Tes!

Ayo tekan bendera hijau! Coba tabrak Zombie!

Apa yang terjadi?

Nyawanya berkurang! ... TAPI KOK CEPAT SEKALI?
(Nyawanya bisa langsung -100 dalam sedetik!)

Jawabannya...

Karena game mengecek 60 kali per detik!
Selama Player masih menyentuh Zombie, nyawanya akan berkurang TERUS MENERUS.

Di game profesional, setelah tertabrak, karakter biasanya 'kebal' sesaat. Ayo kita tiru!

Misi 3: Efek 'Kebal' (Invincibility)

Kita akan buat karakter BERKEDIP setelah tertabrak.
Ini memberi efek visual dan memberi jeda agar nyawa tidak habis instan.

Langkah 4: Kode Berkedip

Kita pakai repeat (Ulangi) dari kategori Control.
Tambahkan ini di dalam if, TEPAT DI BAWAH change [Nyawa] by -1.

Mari Kita Tes (Lagi)!

Tekan bendera hijau! Tabrak Zombie! Bagaimana sekarang?

Hasilnya:
1. Player menyentuh Zombie.
2. Nyawa berkurang 1 (misal: jadi 2).
3. Karakter berkedip-kedip (kebal sesaat).
4. Pemain bisa kabur dulu.

Jauh lebih baik!

Masalah Baru!

Hmm, ada masalah baru...
Playernya sudah aman dan berkedip... TAPI ZOMBIE-NYA KOK TIDAK HILANG?

Zombie yang sudah menyentuh Player harusnya hilang (atau mundur), kan?

Di Zombie!

Tantangan BONUS! (Inovasi)

Tampilan nyawa kita sekarang cuma berupa angka. Agak membosankan!
Bagaimana kalau kita buat visual?

Tantangan Ide:
Buat 3 sprite baru (Hati1, Hati2, Hati3). Saat nyawa berkurang (change [Nyawa] by -1), tambahkan blok broadcast [Nyawa Berkurang]. Setiap sprite Hati menerima broadcast itu dan hide satu per satu. (Ini konsep level lanjut!)

Apa yang Kita Pelajari?

Variabel Lanjutan: Menyimpan status 'Nyawa'.

Set vs Change: Kapan harus mengatur, kapan harus mengubah.

Deteksi Tabrakan: Menggunakan if <touching...?>.

Efek Kebal: Membuat Player berkedip dengan repeat, hide, show, wait.

Logika Terpisah: Logika karakter (berkedip) ada di Karakter, logika rintangan (hilang) ada di Rintangan.
